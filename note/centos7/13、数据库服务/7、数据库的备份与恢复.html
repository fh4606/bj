07-数据库备份与恢复

风华小于 1 分钟

1、将数据库zxw导出到/opt目录下，命令格式： mysqldump -u 【用户名】 -p 【数据库名】 > /opt/zxw.dump

[root@host-172-16-0-4 ~]# mysqldump -u root -p zxw > /opt/zxw.dump

提示输入密码，请输入管理员root密码

2、进入到数据管理系统中，删除库zxw ；命令格式： drop database 【库名】；

MariaDB [(none)]> drop database zxw;

此时查询数据库：

MariaDB [(none)]> show databases;

我们发现已经没有zxw这个库了

3、新建数据库zxw ，命令格式： create database 【库名】；

MariaDB [(none)]> create database zxw;

查询数据库zxw中有哪些表单：命令格式： show tables；

MariaDB [zxw]> show tables;

当前库中没有任何表

4、我们将/opt目录下的zxw导入数据库管理系统中，命令格式： mysql -u 【用户名】 -p 【数据库名】 < /opt/zxw.dump

[root@host-172-16-0-4 ~]# mysql -u root -p zxw < /opt/zxw.dump

此时查询zxw库中的表单，命令格式： show tables；

MariaDB [zxw]> show tables;

此时我们发现读了一个ceshi表

接着查询ceshi表中的数据，使用命令： select * from 【表名】；

MariaDB [zxw]> select * from ceshi;

此时我们发现之前删除的数据又回来了
